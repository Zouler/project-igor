# Visual identity — governance and approval

**Evolving visual bible:** The team’s consolidated direction (pillars, I.G.O.R., world, UI, materials, do/don’t) now lives in **[VISUAL_IDENTITY.md](./VISUAL_IDENTITY.md)**. Treat that file as the **v1 visual bible** to update as decisions are ratified.

This page stays as a **short reminder**: major asset work and aesthetic implementation prompts should still be **debated and approved** before execution.

---

## Before implementing a final visual style, confirm alignment with VISUAL_IDENTITY.md on

- **I.G.O.R.** — shape language and silhouette  
- **Motorlings / Spark Cores** — design vocabulary and readability at small sizes  
- **World palette** — planet, workshop, test zones, UI harmony  
- **UI style** — typography, panels, buttons, child-friendly scale  
- **Toy-like vs mechanical detail** — how “inventor toy” vs “worn machine” reads for ages 5+  
- **Oxidation / worn-metal rules** — decorative charm vs clarity  
- **Child-friendly readability** — contrast, motion, clutter limits (especially on mobile)  

---

## Implementation rule

**No major aesthetic pass or art-direction prompt should be executed in code or assets until this direction is approved.** Small readability tweaks that do not lock art direction may still be documented elsewhere (e.g. [DEV_CHECKLIST.md](./DEV_CHECKLIST.md)).
