# Visual identity — intentionally pending

**Visual identity for Proyecto I.G.O.R. is not defined in this repository yet.** The current build uses placeholder primitives and simple materials so gameplay and flow can ship as MVP.

## Before implementing a final visual style, the team should agree on

- **I.G.O.R.** — final shape language and silhouette
- **Motorlings / Spark Cores** — design vocabulary and readability at small sizes
- **World palette** — planet, workshop, test zones, UI harmony
- **UI style** — typography, panels, buttons, child-friendly scale
- **Toy-like vs mechanical detail** — how “inventor toy” vs “worn machine” reads for ages 5+
- **Oxidation / worn-metal rules** — if used, how far to push grime vs clarity
- **Child-friendly readability** — contrast, motion, clutter limits (especially on mobile)

## Implementation rule

**No major aesthetic pass or art-direction prompt should be executed in code or assets until this direction is approved.** Small readability tweaks that do not lock art direction may still be documented elsewhere (e.g. DEV checklist).
