# Gameplay vision — Proyecto I.G.O.R.

High-level intent for **how the game should feel and loop** beyond the current MVP build.

---

## Main concept

The **child** builds **living helper machines** by giving **bodies** to **Motorlings / Spark Cores**, then **deploys** those machines into the world to:

- Repair terrain  
- Collect materials  
- Build civilization  
- Eventually **rebuild the child’s spaceship**  

Machines are companions with intent, not inert tools.

---

## Main loop (long-term target)

1. **Discover or hatch** a Motorling.  
2. **Build a machine body** in the **Laboratory / Workshop**.  
3. **Test or deploy** the machine.  
4. The machine **performs a task** — succeeds partially, fails **playfully**, or behaves in a **funny** way.  
5. **The world changes** (terrain, props, mood, lighting).  
6. **Materials / resources** are gained.  
7. The child **builds structures** in the **Construction Area**.  
8. **New facilities** unlock new parts and, over time, **spaceship components**.

The current MVP implements an **early slice** of steps 2–4 with scripted test zones and simplified world feedback.

---

## Two main places

### A. Laboratory / Machine Workshop

- **Experimental machine creation** — try combinations.  
- **Motorlings receive bodies** — emotional beat when a Spark Core “fits.”  
- The child **discovers machine types** through parts, not only through menus.  
- **Weird / random machines are allowed** — the world should react with curiosity, not hard failure.

### B. Construction Area / Building Yard

- The child builds **houses, bridges, facilities, paths**, and other **civilization structures**.  
- Building uses **wooden blocks / LEGO-like block logic** (readable stacking, snapping, or grid — to be designed).  
- Structures **help grow** I.G.O.R.’s world and unlock downstream systems.

---

## Tablet-first design

- Designed for **fingers**, not only mouse.  
- **Large touch targets** and forgiving hit areas.  
- **Drag/drop** and **tap** interactions as primary verbs.  
- **Immediate feedback** — motion, sound, I.G.O.R. reactions (when systems exist).  
- **Little text** in moment-to-moment play; rely on visuals and I.G.O.R.  
- **I.G.O.R. reacts often** — success, surprise, gentle coaching.

---

## Relation to MVP

Today: **Workshop + Mission Select + Test Zones + Community** prototype the **workshop → test → world reacts** slice without full construction or deployment simulation. This document describes the **north star** for expansion.
